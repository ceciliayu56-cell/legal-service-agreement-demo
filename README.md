# 法律服务协议线上化 Demo

这是一个前端静态原型，用于演示“法律服务协议合规管控系统”：围绕待用印版本、底线突破申请、用印前强卡点、归档核查和合规抽查后台，帮助律所管理层监控最终对外签署的法律服务协议是否突破事务所底线规则。

## 直接打开

推荐直接打开：

```text
index.html
```

`index.html` 与 `demo.html` 内容一致，均为当前最新版静态 demo，不需要安装依赖，不需要启动服务。

## 当前流程

A版标准模板底稿制作 → 上传B版待用印版本 → 签约确认人责任确认 → 发起用印申请前最终合规校验 → 上传C版 / 归档确认 → 合规事后抽查

## 已实现页面

1. 协议合规工作台
2. A版标准模板底稿制作

5. 发起用印申请前最终合规校验
6. 上传C版 / 归档确认
7. 合规事后抽查后台

## 关键规则

### A版制作页

- A版仅为事务所标准模板底稿，用于后续与客户沟通。
- A版阶段提供填写项、选择项、可删除项识别和底线关注轻提示。
- 如预计突破事务所底线规则，可发起底线突破申请。
- A版不做审批阻断，不展示复杂风险报告。

### B版待用印版本上传页

- B版为最终待用印版本，是用印前合规校验的主要依据。
- 普通用户侧只展示文件信息、底线突破申请状态和下一步动作。
- 复杂A/B差异、条款级疑似风险、AI识别报告进入合规后台展示。

### 签约确认人承诺页

- 不再使用多个 checkbox。
- 页面展示一段统一承诺声明。
- 签约确认人只需点击一次“确认并承诺，进入用印”。
- 签约确认页只做责任确认，不做复杂审查，也不阻断流程。
- 即使底线突破申请为无申请、审批中、未通过或已撤回，仍可完成确认；真正校验在发起用印页。

### 发起用印申请页

- 发起用印页是唯一强卡点。
- 无申请或已撤回：可发起用印，但需确认当前版本不存在底线突破事项。
- 已通过：可发起用印，并关联审批记录。
- 审批中或未通过：阻断用印。

### 上传C版 / 归档确认页

- 用于上传实际盖章版本，形成归档及后续抽查依据。
- 普通用户侧仅显示B/C一致性状态：一致、疑似不一致、待核查，以及是否进入合规抽查池。
