# 法律服务协议系统流程说明

本文档用于说明法律服务协议线上化 demo 的系统流程、各页面操作方式及审核关注点。适用对象包括协议制作人、秘书、签约确认人及后续审核人员。

## 一、系统主流程

当前流程为：

制作规则版 / A版 → 上传B版与风险确认 → 签约确认人承诺 → 发起用印申请 → 上传C版 / 归档确认

各节点定位如下：

| 流程节点 | 主要目标 | 责任人 | 关键结论 |
| --- | --- | --- | --- |
| 制作规则版 / A版 | 基于所内标准模板生成协议底稿 | 制作人 / 秘书 | A版是内部模板底稿，不是最终责任确认节点 |
| 上传B版与风险确认 | 上传客户沟通后的待用印版本，并确认风险差异 | 制作人 / 秘书 | B版是风险最终确认节点 |
| 签约确认人承诺 | 签约确认人确认已阅读待用印版本及风险提示 | 签约确认人 | 不重新发起风险申请，只做责任确认 |
| 发起用印申请 | 校验前置流程并生成用印申请信息 | 秘书 / 发起人 | 用印前必须关联案件 |
| 上传C版 / 归档确认 | 上传双方签署/盖章后的最终版本，并确认归档状态 | 秘书 | 形成最终归档文件 |

## 二、制作人如何使用

### 1. 制作规则版 / A版

A版页面用于根据所内标准模板制作协议底稿。

制作人需要完成：

1. 填写协议首页、委托人、世辉主体、日期、协议有效期等挖空字段。
2. 在服务范围、费用及支付方式等条款位置填写大段内容。
3. 对有选项的内容进行选择，例如协议类型、主体类型等。
4. 阅读黑字标准条款，不得直接编辑。
5. 如确需修改黑字标准条款，点击条款旁的“申请修改”，填写修改预报备。

黑字条款修改规则：

1. 黑字标准条款默认锁定，不能直接改成普通输入框。
2. 点击“申请修改”后，填写：
   - 条款位置
   - 原标准条款
   - 拟修改后内容
   - 修改原因
   - 是否预计突破底线
   - 风险等级
3. 保存后先形成“待提交修改”。
4. 点击“合并提交修改预报备”后，状态变为“已提交预报备”。
5. A版预报备不阻断A版生成或下载。
6. A版预报备不自动豁免B版风险，B版阶段仍需确认是否适用。

页面按钮：

| 按钮 | 用途 |
| --- | --- |
| 保存草稿 | 保存当前填写内容 |
| 预览正式版 | 查看隐藏提示和编辑边框后的正式协议效果 |
| 生成A版 | 生成所内模板底稿 |
| 下载A版 | 生成A版后下载底稿 |
| 进入上传B版 | 进入客户沟通后的待用印版本上传节点 |

### 2. 上传B版与风险确认

B版是客户沟通后的待用印版本，是风险确认的关键节点。

制作人需要完成：

1. 上传B版文件。
2. 查看系统生成的 mock 差异报告。
3. 按差异类型逐项处理。
4. 确认不存在待处理风险差异。
5. 点击“确认风险差异，进入签约确认人承诺”。

B版页面中的案件关联信息为可选项：

1. 可以提前关联案件。
2. 未关联案件也可以上传B版。
3. 未关联案件也可以完成B版风险确认。
4. 未关联案件也可以进入签约确认人承诺。
5. 案件信息将在“发起用印申请”前强制校验。

B版差异分三类：

| 差异类型 | 说明 | 是否影响进入下一步 |
| --- | --- | --- |
| 可引用A版预报备差异 | A版阶段已有预报备，但需要确认是否适用于B版最终修改 | 未处理时影响 |
| 需补充突破说明 | 未报备的标准条款修改，或B版内容与A版预报备不一致 | 未填写说明时影响 |
| 普通差异，仅留痕 | 不涉及标准条款修改或底线突破 | 不影响 |

可引用A版预报备差异的处理方式：

1. 如果B版实际修改内容与A版预报备一致，点击“应用已有预报备”。
2. 如果不一致，点击“补充/重新填写说明”。

需补充突破说明的处理方式：

1. 可点击单条差异的“填写突破说明”。
2. 如果存在多条需说明差异，也可在底部“统一突破说明”区域一次性填写。

B版确认按钮启用条件：

1. B版文件已上传。
2. 系统已生成风险/差异结果。
3. 所有可引用A版预报备差异均已处理。
4. 所有需补充突破说明的差异均已填写说明。
5. 不存在待处理风险差异。

### 3. 签约确认人承诺页

该页面用于签约确认人一次性确认风险与版本情况。

签约确认人需要查看：

1. 合同基础信息。
2. 当前B版文件。
3. 风险摘要。
4. 风险明细。
5. 承诺声明。

承诺声明为：

“我确认已阅读本协议待用印版本，并已知悉系统识别的版本差异及风险提示；我确认已就已知突破事项完成报备，且不存在其他未披露的突破事项。”

操作规则：

1. 页面不再要求逐项勾选 checkbox。
2. 点击“确认并承诺，进入用印”即表示签约确认人确认上述承诺内容。
3. 如仍存在未报备标准条款修改或未处理风险，主按钮置灰。
4. 如发现问题，应点击“返回B版风险确认”，补充处理后再提交承诺。

### 4. 发起用印申请

该页面是签约确认人承诺完成后的用印前置节点。

发起人需要确认：

1. B版已上传。
2. 风险确认已完成。
3. 签约确认人承诺已完成。
4. 已关联案件。
5. 不存在未处理风险。

案件关联强校验：

1. 发起用印申请前必须关联案件。
2. 如未关联，系统提示：
   “当前合同尚未关联案件，无法发起用印申请。请先关联案件信息。”
3. 关联案件支持多选。
4. 搜索字段包括案号、案件名称、客户名称、合伙人、经办律师。

用印申请预填信息包括：

| 字段 | 说明 |
| --- | --- |
| 用章性质 | 默认业务类 |
| 业务类用章用途 | 默认法律服务协议 |
| 印章选择 | 根据签约主体/地域默认带出，可修改 |
| 用印文件 | 自动带入当前B版 |
| 盖章份数 | 默认2，可修改 |
| 审批人 | 默认签约确认人 |
| 备注 | 可编辑 |

点击“提交用印申请”后，demo 展示提示：

“用印申请已生成，请在飞书审批中确认并提交。”

真实上线后，该按钮可对接飞书审批接口，预填用印申请表，但最终仍需发起人在飞书中确认提交。

### 5. 上传C版 / 归档确认

C版页面用于上传双方签署/盖章后的最终版本，并确认归档状态。

页面模块顺序：

1. 当前B版 / 待用印版。
2. 合同基础信息。
3. 上传C版 / 双章版。
4. 归档状态确认。

制作人需要完成：

1. 查看当前B版文件信息。
2. 上传C版/双章版文件。
3. 查看上传成功状态。
4. 根据电子归档和纸质归档情况确认归档状态。

归档状态包括：

| 类型 | 状态 |
| --- | --- |
| 电子归档状态 | 未上传C版 / 已上传C版 |
| 纸质归档状态 | 无纸质版，仅电子归档 / 已收到纸质版，待归档 / 已完成纸质归档 |

点击“确认归档”后，合同状态可变为：

1. 已归档。
2. 电子已归档 / 纸质待归档。

## 三、审核人如何审核

审核动作分为三个层次：A版预报备留痕、B版风险确认、用印后归档核查。

### 1. A版阶段审核关注点

A版阶段不是最终合规审查节点，主要用于提前发现和留痕。

审核人关注：

1. 是否存在黑字标准条款修改预报备。
2. 预报备是否说明了客户要求、业务背景和修改原因。
3. 是否标明预计突破底线。
4. 风险等级是否明显偏低。
5. 拟修改内容是否涉及责任限制、费用、账期、保密、利冲等重点条款。

注意：

1. A版预报备不等于审批通过。
2. A版预报备不自动豁免B版风险。
3. 后续应以B版最终内容为准重新确认。

### 2. B版风险确认审核关注点

B版是准备用印版本，是风险确认的核心节点。

审核人关注：

1. B版是否已上传。
2. B版差异是否已生成。
3. 可引用A版预报备差异是否已人工确认。
4. 需补充突破说明的差异是否已填写说明。
5. 是否仍存在待处理风险差异。
6. 统一突破说明是否足以覆盖多条风险差异。
7. 普通差异是否确实不涉及标准条款或底线突破。

对A版预报备差异的审核：

1. 核对标准模板内容。
2. 核对A版预报备拟修改内容。
3. 核对B版实际修改内容。
4. 判断B版内容是否与A版预报备一致。
5. 如一致，可接受“应用已有预报备”。
6. 如不一致，应要求补充或重新填写突破说明。

对需补充突破说明差异的审核：

1. 检查说明是否包含客户要求。
2. 检查说明是否包含业务背景。
3. 检查说明是否说明已与签约确认人沟通。
4. 检查说明是否覆盖实际修改内容。
5. 对高风险条款应重点关注责任上限、赔偿范围、账期、费用减免、保密期限、利益冲突、客户模板底线条款等。

### 3. 签约确认人承诺审核关注点

签约确认人承诺不是风险发现节点，而是责任确认节点。

审核人关注：

1. 签约确认人是否已完成承诺。
2. 承诺前是否仍有未处理风险。
3. 承诺时展示的风险摘要和明细是否完整。
4. 如存在未处理风险，是否返回B版风险确认补充处理。

签约确认人不应在承诺页重新发起新的风险申请。

### 4. 用印申请审核关注点

用印申请前必须完成前置校验。

审核人关注：

1. 是否已关联案件。
2. B版是否已上传。
3. 风险确认是否已完成。
4. 签约确认人承诺是否已完成。
5. 是否仍存在未处理风险。
6. 用印文件是否为当前B版。
7. 印章、份数、审批人、备注是否合理。

如未关联案件，应阻止提交用印申请。

### 5. C版归档审核关注点

C版/双章版是最终归档版本。

审核人关注：

1. C版是否已上传。
2. C版文件是否为双方签署/盖章后的最终版本。
3. 当前B版信息是否可追溯。
4. 电子归档状态是否正确。
5. 纸质归档状态是否正确。
6. 如后续做B/C一致性核查，应关注C版是否与B版存在实质差异。

## 四、关键规则汇总

1. A版是所内模板底稿，不是最终责任确认节点。
2. A版可做黑字条款修改预报备，但不阻断生成和下载。
3. A版预报备不自动豁免B版风险。
4. B版是待用印版本，是风险差异确认节点。
5. B版案件关联为可选，不影响风险确认。
6. 发起用印申请前必须关联案件。
7. 签约确认人承诺只需一次点击确认，不需要逐项勾选。
8. C版用于最终归档确认，支持电子归档和纸质归档状态记录。

## 五、推荐使用顺序

制作人建议按以下顺序操作：

1. 进入制作规则版/A版。
2. 完成字段填写和正文编辑。
3. 对确需修改的黑字条款保存并提交预报备。
4. 生成并下载A版。
5. 客户沟通后上传B版。
6. 在B版页面处理全部风险差异。
7. 进入签约确认人承诺页。
8. 签约确认人完成一次性承诺。
9. 发起用印申请，并在此节点完成案件关联。
10. 用印完成后上传C版/双章版。
11. 确认电子和纸质归档状态。

审核人建议按以下顺序审核：

1. 先看B版风险差异，而不是只看A版预报备。
2. 核对A版预报备是否真正适用于B版最终修改。
3. 检查所有需补充说明的差异是否已处理。
4. 检查签约确认人承诺是否在风险处理后完成。
5. 检查用印申请是否具备案件关联和完整前置状态。
6. 检查C版归档信息是否完整。
